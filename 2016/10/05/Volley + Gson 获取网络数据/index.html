Volley + Gson 获取网络数据

Posted on 2016-10-05 | In Android

在百度上看到了查询城市公交车的Api接口，所以想写一个App，只写了一点试试获取数据，用到了 Volley + Gson 的方式拿到数据，写在这里做一个总结。

http://apistore.baidu.com/apiworks/servicedetail/1697.html

需要用到的

首先需要导入 Volley 和 Gson 的库

https://github.com/google/gson

http://download.csdn.net/detail/sinyu890807/7152015

既然要访问网络，不要忘了添加网络权限

<uses-permission android:name="android.permission.INTERNET"/>

项目结构

我用的是 MVP 模式，这种模式可以把以前在 Activity 的大量代码放在 Presenter 里面，只是多了些接口。

Getting Start

首先新建 MyApplication 

public class MyApplication extends Application {

    private  VolleyHttpClient volleyHttpClient;
    private static MyApplication instance;

    public static synchronized MyApplication getInstance(){
        if (instance==null){
            instance = new MyApplication();
        }
        return instance;
    }
    public VolleyHttpClient getVolleyInstance(Context context){
        if (volleyHttpClient==null){
            volleyHttpClient = new VolleyHttpClient(context);
        }
        return volleyHttpClient;
    }
}

使用单例模式创建 Application 实例和 VolleyHttpClient ，VolleyHttpClient 是哪来的？它是我创建的一个类，用于创建 Volley 队列，和加入队列的方法。下面贴出代码。

public class VolleyHttpClient {

    private RequestQueue mRequestQueue;
    public VolleyHttpClient(Context context){
        this.mRequestQueue = Volley.newRequestQueue(context.getApplicationContext());
    }
    public <T> void addToRequestQueue(Request<T> res){
        res.setRetryPolicy(new DefaultRetryPolicy(20*1000,1,1.0f));
        mRequestQueue.add(res);
    }
}

Volley的使用流程是，首先创建一个队列，然后在中间写获取数据的代码，然后在下面把请求加入队列。

接下来，我们要自定义 Request 

public class VolleyHttpRequest<T> extends Request<T> {

    private Response.Listener<T> mListener; 
    private final Type classType; //Gson需要传入classType
    private Gson gson;
    public VolleyHttpRequest(int method,String url,Type classType, Response.Listener<T> listener, Response.ErrorListener errlistener) {
        super(method, url, errlistener);
        gson = new Gson();
        mListener = listener;
        this.classType = classType;
    }

    @Override
    public Map<String, String> getHeaders() throws AuthFailureError {
        Map<String,String> map = new HashMap<>();
        map.put("apikey","891eeffe6f1b4652128542bb8c9971a2");
        return map;
    }

    public VolleyHttpRequest(String url,Type classType, Response.Listener<T> listener,Response.ErrorListener errlistener){
        this(Method.GET,url,classType,listener,errlistener);
    }

    @Override
    protected Response<T> parseNetworkResponse(NetworkResponse response) {
        T resultObject;
        try {
            String jsonString = new String(response.data, HttpHeaderParser.parseCharset(response.headers));
            resultObject = gson.fromJson(jsonString,classType);
            return Response.success(resultObject,HttpHeaderParser.parseCacheHeaders(response));
        } catch (UnsupportedEncodingException e) {
            e.printStackTrace();
        }
        return null;
    }

    @Override
    protected void deliverResponse(T response) {
        mListener.onResponse(response);
    }
}

自定义 Request 需要覆写两个方法， parseNetworkResponse（）和 deliverResponse（），其中 deliverResponse 方法很简单，仅仅是调用了mListener 的 onResponse（）方法，传入 response 即可。getHeaders（）方法的作用是在信息头写入一些信息，不是必须的。parseNetworkResponse（）是对服务器响应的数据进行解析，其中的数据存放在 NetworkResponse 的 data 变量中，这里用到了 Gson，简单的一行代码就把 json 数据解析出来，并传入 Response 的 success（）方法。

新建 BasePresenter 类

public class BasePresenter {

    public Context context;
    VolleyHttpClient httpClient;
    BasePresenter(Context context){
        this.context = context;
        httpClient = MyApplication.getInstance().getVolleyInstance(context);
    }
}

新建 BusPresenter 类

public class BusPresenter extends BasePresenter {


    public BusPresenter(Context context) {
        super(context);
    }

    public void requestBusData(final BusInterface callback) {
        Map<String, Object> params = new HashMap<>();
        params.put("city", "杭州");
        params.put("bus", "195");
        params.put("direction", "0");

        String url = GetParamsFormatter.format(Constant.URL, params);

        Type type = new TypeToken<Bus>() {
        }.getType();
        VolleyHttpRequest<Bus> request = new VolleyHttpRequest<Bus>(url, type, new Response.Listener<Bus>() {
            @Override
            public void onResponse(Bus buses) {
                callback.onBusDataRequestSuccess(buses);
            }
        }, new Response.ErrorListener() {
            @Override
            public void onErrorResponse(VolleyError volleyError) {

            }
        });

        httpClient.addToRequestQueue(request);

    }
}

这个类继承自 BasePresenter 类，所以在构造函数里调用 super(),就调用了 BasePresenter 的构造方法,创建了 VolleyHttpClient 实例，再看 VolleyHttpClient 的构造方法创建了队列。这个时候队列就创建好了，就可以在中间写获取数据的部分了。上面说使用 Presenter 多了一些接口就在这里，BusInterface callback 这个接口是我自己创建的，目的是把数据传递到需要的地方。哪需要就实现这个接口就好了。在这里我把数据传递到MainActivity里面。

public class MainActivity extends AppCompatActivity implements BusInterface {

    private BusPresenter presenter;
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        presenter = new BusPresenter(this);
        presenter.requestBusData(this);
    }


    @Override
    public void onBusDataRequestSuccess(Bus bus) {
        Log.i("tyq", bus.getData().getStations().get(0).getStateName()+"");
    }
}
